# ResearchSense

A research information system for **Bahria University Islamabad, E-8 campus** —
researcher profiles, publications, research areas, funded projects, a
collaboration finder, and a research assistant. Inspired by the Elsevier Pure
portal layout (research.nu.edu.kz), with Bahria's own visual identity.

Built as a Final Year Project. **Phase 1** ships the full UI plus a FastAPI
backend serving mock JSON built from the real Bahria E-8 Computer Science
faculty roster. The data layer is isolated behind repository interfaces so
moving to a real database is a drop-in change.

## Architecture

```
Frontend (React + TS)  ──HTTP──►  FastAPI
  api/ (typed clients)              routers/   thin HTTP layer
  types/ (mirror schemas)           services/  business logic
  components/ features/ pages/      repositories/  data-access interface
                                      mock/ (JSON now) → SQL (later)
                                    schemas/   Pydantic contract
                                    data/      seed JSON
```

- Components never call `fetch` directly — only `src/api/*`.
- Routers hold no logic; repositories are the only code touching the data source.
- Every source file is kept under ~350 lines, one responsibility each.

## Run it

**Backend** (Python 3.11+):

```bash
cd backend
python -m venv .venv
.venv/Scripts/activate        # Windows;  source .venv/bin/activate on macOS/Linux
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000
```

API docs: http://127.0.0.1:8000/docs

**Frontend** (Node 18+):

```bash
cd frontend
npm install
npm run dev                   # http://127.0.0.1:5173
```

The Vite dev server proxies `/api` to the backend, so no CORS setup is needed.

## Data

Seed JSON lives in `backend/app/data/`. It is generated by:

```bash
cd backend
python -m scripts.build_seed
```

- **Real** data: researcher names, designations, department, and (for senior
  faculty) research areas — from the published Bahria E-8 roster.
- **Derived / sample** data: publications, projects, funding, emails, ORCIDs —
  each record carries `source: "sample"` to keep provenance honest.

To re-pull the live roster (needs Playwright + Chromium), run
`python -m scripts.scrape_bahria`, then `build_seed`.
